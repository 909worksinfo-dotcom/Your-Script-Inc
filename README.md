# 🎬 AI Agent 短剧剧本工作室 (Script Studio)

一个由 **工作室管理者 + 5 位 AI Agent 创作数字员工**组成的可视化短剧剧本工作室。管理者负责调度、验收与返工闭环，5 位创作数字员工分工协作、依次接力，调用《三幕式创意和剧本生成》工具的 Prompt 与 LLM 服务，从一句原始创意出发，**全流程可视化**地产出一部高质量的海外短剧剧本（三幕式创意 → 分集大纲 → 分镜脚本表格 → 飞书交付文档）。

> 默认使用「Mock (演示)」模式，**无需任何 API Key 即可离线走通全流程与可视化**；需要真实生成时，在侧边栏选择真实服务商并填入 API Key。当前版本重点强化了长流程稳定性：后台执行、跨刷新保留、断网长重试、任务 8 批次完整性保护和本地受控部署。

---

## ✨ 核心特性

- **工作室管理者 + 5 位创作数字员工**：工作室管理者🧭负责任务调度、质量验收与返工闭环；短剧爆款研究员🛰️、短剧创意天才💡、经验丰富的短剧编剧🎬、犀利的短剧剧本审核员⚖️、文档助理📋负责具体创作交付。角色介绍与工作技能均严格采用产品需求文档原文，并配备 Agent 技能与记忆。
- **9 大协作任务**：原始创意 → 三幕式创意 → 创意审核 → 创意精修 → 分集大纲 → 大纲审核 → 大纲精修 → 分镜脚本 → 飞书归档。依赖关系严格编排，审核员意见会作为精修任务的前置输入，每个任务产出后由管理者验收，不合格会自动带意见返工。
- **两种运营方式**：
  - **① 自动按序执行**：一键运行全流程，5 位员工依次自动完成 9 个任务。
  - **② 手动逐步启动**：逐个任务点击执行；每步产出可**修改**后再进入下一步，也可**跳过本任务直接粘贴本地剧本文本**；还可随时从当前进度**一键自动跑完后续剩余任务**（不回头补跑前序）。
- **完全可视化**：像素风办公室全景 + 5 位数字员工卡通形象坐在各自工位，正在工作的员工会**跳动并高亮**；下方是 9 大任务的实时协作流水线。
- **逐员工模型配置**：每位员工可独立配置 API 服务商 / API Key / 模型（选项与原工具完全一致）；也支持全局默认。
- **长流程稳定执行**：自动全流程与“自动执行后续剩余任务”均在后台线程执行，产出、日志、进度写入跨会话 `RunStore`，浏览器刷新、重连或短暂断开后仍可从当前状态继续查看。
- **断网恢复能力**：真实模型调用遇到断网、超时、429、5xx、连接中断等临时错误时，会进入长重试（最多 15 次，累计等待约 47 分钟），网络恢复后继续当前任务或任务 8 当前批次；鉴权、参数、权限、内容拦截、额度等永久错误不会无意义重试。
- **任务 8 完整性保护**：分镜脚本按批次生成，只跳过已成功批次；失败批次会在恢复后重新生成。任务 8 只有在目标批次完整、无失败内容、集数标记覆盖目标集数时才允许进入任务 9。本地完整粘贴分镜仍可作为手动跳过方式。
- **质量校验**：分集大纲集数完整性与每集 300-500 字校验、分镜 CSV 解析与批次状态诊断、飞书文档自动转 Markdown 表格。

## 🚀 本地运行

要求 Python 3.9–3.13。

推荐使用本地部署管理器启动。它不会改变 UI 或业务功能，只是把裸 `streamlit run` 包装成更稳定、可管理的本地服务：自动避开端口冲突、后台启动、健康检查、记录 PID 和日志，并在 macOS 上用 `caffeinate` 降低运行中睡眠导致中断的概率。

```bash
# 1) 安装依赖
python3 deploy.py setup

# 2) 后台启动并打开浏览器
python3 deploy.py start --open

# 3) 查看状态 / 日志
python3 deploy.py status
python3 deploy.py logs

# 4) 停止 / 重启
python3 deploy.py stop
python3 deploy.py restart --open
```

启动后浏览器访问命令输出的本地地址。默认从 `http://localhost:8520` 开始自动寻找空闲端口；如果 8520 已被占用，会自动尝试后续端口。

运行时文件位于 `.script_studio_runtime/`，包含 PID、端口和日志；该目录已加入 `.gitignore`。如果服务异常，优先查看：

```bash
python3 deploy.py status
python3 deploy.py logs -n 120
```

如需调试 Streamlit 本身，仍可使用原始方式前台启动：

```bash
python3 -m streamlit run script_studio.py
```

> 提示：修改了 `studio/` 里的任何文件后，需要**重启服务**才会生效（Streamlit 会缓存已导入的包模块）。

## 🤖 使用说明

1. **选服务商**（侧边栏「模型配置」）：默认 `Mock (演示)` 可离线演示；真实生成请选 Azure OpenAI (ByteDance) / OpenRouter / Google Gemini / OpenAI (GPT) / Anthropic (Claude) 并填入 API Key。
2. **项目设置**：设置剧本总集数（默认 50）、分镜模式（标准短剧 / 解说漫）。
3. **选运营方式**：
   - 自动：点「🚀 一键运行全流程」。
   - 手动：逐个任务点「启动该任务」；可在每个任务的「查看 / 修改 / 粘贴」面板里改产出或粘贴本地文本；也可点「🚀 自动执行后续剩余任务」从当前进度往后自动跑完。
4. **下载交付物**：各任务产出可下载 TXT；分镜可下载 CSV（Excel 专用）；最终飞书文档可下载 Markdown。

## 🛡️ 稳定性设计

### 1. 浏览器断连 / 刷新不丢进度

Streamlit 的页面脚本与浏览器会话绑定，普通 `streamlit run` 在长任务、电脑休眠或浏览器断开时容易丢失前台状态。当前版本将长流程运行数据迁移到 `studio.store.RunStore`：

- 产出、运行进度、员工记忆、管理者验收记录、运行日志保存在跨会话单例中。
- 自动全流程在后台线程执行，不依赖 `st.session_state`。
- 刷新或重连后，页面从 `RunStore` 重新读取进度与产出。

### 2. 断网后自动等待恢复

真实模型调用遇到临时网络/API 错误时，系统不会立即失败，而是写入运行日志并按长间隔等待重试：

- 会重试：断网、DNS/连接错误、超时、连接重置、429、500/502/503/504、服务暂时不可用。
- 不重试：API Key 错误、权限错误、参数错误、模型不存在、内容拦截、余额/额度问题。
- 用户点击“停止自动执行”或“重置工作室”时，等待中的重试会被打断。

### 3. 任务 8 分镜批次保护

任务 8 是分批生成，当前版本对批次状态做了专门保护：

- 已成功批次会跳过，避免重复消耗。
- 已失败或无效批次会在恢复后重新生成。
- 自动进入任务 9 前会再次检查任务 8：批次完整、无 `❌` 错误、集数覆盖目标集数。
- 运行日志会显示成功批次、失败批次、缺失批次、本地粘贴批次和集数标记数量，便于排查。

## ☁️ 部署到 Streamlit Community Cloud

1. 将本目录推送到 GitHub 仓库（确保 `script_studio.py`、`studio/`、`requirements.txt`、`studio_office.png` 都在仓库根目录）。
2. 打开 [share.streamlit.io](https://share.streamlit.io) → New app → 选择仓库、分支，**Main file path 填 `script_studio.py`** → Deploy。
3. 部署后默认即为 Mock 演示，访问者可在侧边栏自行填入各自的 API Key 进行真实生成。

**注意事项**
- 仓库中**不含任何密钥**（Key 由侧边栏手动输入），可放心公开。
- `deploy.py` 是本地稳定部署管理器，Streamlit Community Cloud 仍然使用 `script_studio.py` 作为 Main file path。
- `requirements.txt` 当前未锁定版本，云端会安装最新依赖；如需更稳定的构建，建议固定版本（其中 `google-generativeai` 已被官方标记弃用，若不使用 Gemini 可移除——代码为容错导入，移除后不影响其它服务商）。
- `max_tokens` 设为 60000（见 `studio/llm_service.py` 的 `MAX_TOKENS`），需所选模型支持大输出；旧版 `claude-3-x`（最大 4096–8192）会被 API 拒绝并返回友好报错，换用支持大输出的模型即可。
